# dimens-report 使用分层说明

## 1. 文档目标

这份文档负责解释 `dimens-report` 的业务分层，回答下面几个问题：

1. 报表主资源、组件、查询、参数联动分别是什么层。
2. 为什么“图表空白”不一定是没数据。
3. 为什么报表问题要同时看项目权限、参数、数据源和组件配置。

它不代替接口案例，也不代替能力状态说明。

---

## 2. 四层业务分层

### 2.1 报表主资源层

主资源层负责回答：

- 项目里有哪些报表
- 报表详情、配置、生命周期状态是什么
- 报表是否发布、归档、移动或复制

### 2.2 组件层

组件层负责回答：

- 报表里有哪些图表组件
- 组件类型、布局、数据映射和图表配置是什么
- 为什么某个组件不显示或顺序变化
- 当前组件类型是否属于前端真实支持的 `recharts@3.6.0` 渲染范围

### 2.3 查询层

查询层负责回答：

- 整个报表查询如何执行
- 单个组件如何查询
- 数据预览为什么有数据，正式组件却没有

### 2.4 参数与数据源层

参数与数据源层负责回答：

- 参数默认值、联动关系是否正确
- 数据源类型是什么
- 参数是否真正传到查询层
- 多维表、SQL、API 等不同数据源的限制是什么
- 当数据源来自多维表格时，`recommendedMapping`、`previewMapping` 和最终 `dataMapping` 是否同时完整

---

## 3. 默认判断顺序

遇到报表问题时，建议固定按下面顺序判断：

1. 先确认 `projectId + reportId`。
2. 再判断是主资源问题、组件问题还是查询问题。
3. 如果是组件生成或渲染问题，先核对图表类型与 `dataMapping`。
4. 如果图表空白，再继续看参数和数据源。
5. 如果不同人结果不同，再继续看项目权限和底层数据源权限。

如果问题是“AI 直接生成报表为什么失败”，还要额外判断是否跳过了固定预检链：

1. 是否先创建了主资源
2. 是否先跑了 `preview`
3. 是否在创建组件后跑了 `query-widget`
4. 是否最后再跑整报表 `query`

---

## 4. 为什么图表空白不一定是没数据

图表空白至少有四种常见原因：

1. 参数默认值不对
2. 参数联动条件没满足
3. 数据源有数据，但查询层没拿到
4. 查询有结果，但组件映射配置有问题
5. AI 生成时漏掉了 `sheet.columns`、`fieldIds`、`previewMapping` 等关键层

因此 Skill 不能直接把“空白”翻译成“无数据”。

也不能把“空白”直接翻译成“前端渲染问题”，因为很可能是固定预检链前半段就已经出错。

## 5. 与其他 references 的分工

- `examples.md`：接口级案例总览，重点是主资源、组件、查询、模板、版本接口。
- `capability-status.md`：解释当前报表哪些能力已 CLI 化、哪些仍未完全对齐。
- `recharts-widget-guide.md`：解释前端真实支持哪些组件，以及多维表格报表组件如何一次生成成功。

如果要防止 AI 跑偏，建议把本文件和 `recharts-widget-guide.md` 结合看，不要只看命令案例。

---

## 6. 推荐阅读顺序

建议按下面顺序组合使用：

1. 先看本文件，确认问题属于哪一层。
2. 再看 `examples.md`，确认真实接口路径、方法、入参出参。
3. 如果问题涉及组件生成、字段映射或 Recharts 行为，再看 `recharts-widget-guide.md`。
4. 如果问题是“CLI 到底有没有这能力”，再看 `capability-status.md`。
