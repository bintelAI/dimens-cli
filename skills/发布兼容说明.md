# skills 发布兼容说明

## 1. 目的

`dimens-cli/skills` 这套技能最初按项目内独立知识体系维护，已有较稳定的 `references/` 文档组织方式。

为了兼容 ClawHub / OpenClaw 等技能平台的发布目录规范，本目录采用以下策略：

- 保留原有 `references/`，避免打断现有技能内部引用
- 在各技能目录补充 `README.md`
- 在各技能目录补充 `rules/` 兼容入口
- 在各技能目录补充 `assets/` 占位目录，保证发布包结构完整

## 2. 当前兼容策略

### 2.1 `SKILL.md`

每个技能目录都已补齐：

- `name`
- `slug`
- `description`
- `version`
- `author`
- `tags`

### 2.2 `README.md`

每个技能目录都已补充：

- 技能简介
- 适用场景
- 快速开始
- 目录说明
- 参考资料

### 2.3 `rules/`

`rules/` 目录作为平台兼容层使用，不直接替代原始 `references/`。

推荐做法：

- `rules/README.md` 作为入口说明
- 明确指出业务正文优先看当前技能目录下的 `SKILL.md` 与 `README.md`
- 需要接口级细节时，再进入当前技能目录下的 `references/`
- 后续如果平台确实要求规则文件必须直接放在 `rules/` 下，再做镜像或拆分

### 2.4 `assets/`

`assets/` 当前统一保留 `README.md` 占位文件，并补充统一素材命名规范，目的有三个：

- 兼容技能平台对资源目录的存在性要求
- 为后续技能封面图、图标、截图预留稳定目录
- 统一封面、图标、截图的文件命名与推荐尺寸

当前统一推荐文件名：

- `cover.png`
- `icon.png`
- `screenshot-01.png`
- `screenshot-02.png`
- `screenshot-03.png`

通用规范见：

- `assets-命名规范.md`

## 3. 相对路径兼容策略

为了降低平台对 `../...` 相对链接支持不稳定带来的风险，当前采用以下约束：

- 跨技能导航优先写成“进入某个技能目录，再看其中的 `references/*.md`”
- `rules/README.md` 不再依赖 `../SKILL.md` 或 `../references/...`
- 根目录 `README.md` 负责说明整套 Skill 体系目录结构

这样即使平台不保留原始相对链接，使用者仍然可以按技能目录名手动进入对应文档。

## 4. 后续建议

如果平台后续增加更严格的审核项，建议继续补充：

- 每个技能的封面图到 `assets/`
- 每个技能的图标到 `assets/`
- 统一的发布清单与截图
- `rules/` 下更细粒度的规则拆分文件
