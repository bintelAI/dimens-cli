# dimens-cli/skills 端到端发布检查清单（2026-04-20）

检查日期：2026-04-20  
检查范围：`dimens-project`、`dimens-table`、`dimens-permission`、`dimens-system-orchestrator`  
检查目标：确认“创建项目 -> 创建表 -> 创建字段 / relation -> 创建角色 -> 配项目权限 -> 分配角色”这条技能主链，在发布到 ClawHub/OpenClaw 前是否已经达到可直接复用状态。

---

## 1. 状态口径

本清单统一使用三种状态：

| 状态 | 含义 |
| --- | --- |
| `已命令级验证` | 已有对应 CLI 命令，且当前仓库内已有测试直接覆盖这条用法 |
| `仅文档覆盖` | 文档里已经明确写法或限制，但当前还没有直接测试覆盖 |
| `当前不支持` | 当前 CLI 未封装，或能力仅能依赖 API / server-only 结构 |

---

## 2. 主链总览

当前推荐主链：

1. `dimens-team`
2. `dimens-project`
3. `dimens-table`
4. `dimens-permission`

面向其他产品接技能时，推荐优先使用：

- `--app-url https://dimens.bintelai.com/#/TEAM_ID/PROJECT_ID/`

原因：

- CLI 已支持自动解析 `teamId / projectId / baseUrl`
- 当前主链里的关键命令已有命令级测试覆盖
- 可以减少其他产品手动拆上下文的出错率

## 5. 当前发布结论

面向“其他产品直接使用技能一步到位”的目标，当前可以下这个结论：

- `dimens-project / dimens-table / dimens-permission / dimens-system-orchestrator` 这四个核心技能，主链已经达到可发布状态
- 对外最稳的执行方式是：用户给维表页面地址，技能统一优先生成 `--app-url` 写法
- 当前剩余风险已经从“主链不通”收敛成“复杂 relation 组合回归不足”和“row-acl 非主链撤销未封装”
