# dimens-manager 多维表格章节 字段与系统视图规则

## 1. 这份文档解决什么问题

用于补充 `dimens-manager/references/table/overview.md` 主 Skill 中的系统视图规则与字段规则总引用，避免：

- 把团队级默认能力和项目级实际分配混淆
- 把字段设计、筛选设计和系统视图规则混在一份文档里，导致 Skill 不好引用

字段设计细则和 `row/page` 筛选细则已经拆到单独 references：

| 文档 | 职责 |
| --- | --- |
| `field-design-patterns.md` | 字段设计模板、业务字段分层、relation 设计 |
| `field-option-colors.md` | 单选/多选/下拉选择器颜色协议、默认内置色、自定义色 |
| `row-filters.md` | `row/page` 的搜索、筛选、排序、`viewId` 继承案例 |
| `examples.md` | 接口级方法、路径、CLI 命令、返回结构与实测案例 |

## 2. 当前必须记住的系统视图口径

- `usageType` 是团队级默认能力类型
- `systemView` 是项目级实际系统视图分配
- 项目绑定优先，全局默认回退

## 3. 为什么这个区别重要

如果不区分这两层，会出现：

- 团队里工作流明明存在，但项目里入口不对
- AI 分析、审批、自动化入口显示错位
- 把“默认回退”误判成“项目已绑定”

## 4. 对提示语的要求

遇到系统视图问题时，建议固定提醒：

1. 先确认团队默认类型
2. 再确认项目实际分配
3. 最后判断当前业务入口读取的是哪一层数据

## 5. 字段设计与 `row/page` 的关系

字段不是孤立配置，它直接决定 `row/page` 好不好用，但字段设计细节已经单独拆走，这里只保留总原则：

| 查询需求 | 更适合的字段类型 / 设计 | 说明 |
| --- | --- | --- |
| 关键词搜索 | `text` 主字段、名称字段、编号字段 | 通常配合 `keyword + searchFieldIds` |
| 状态筛选 | `select` / 枚举字段 | 适合 `equals` / `not_equals` |
| 时间区间筛选 | `date` / `datetime` | 适合做区间、前后比较 |
| 金额排序 | `number` | 适合 `sortRule` |
| 跨表筛选 | `relation` | 适合按目标行 ID 或目标对象过滤 |

补充：

- 如果状态字段要在前端以标签形式展示，字段设计时不要漏掉 `color`
- 颜色策略统一参考 `field-option-colors.md`
- 当前前端已经支持内置颜色和 `custom:` 自定义颜色两种协议

需要细化案例时：

1. 先看 `field-design-patterns.md`，决定字段如何设计。
2. 再看 `row-filters.md`，决定查询如何表达。
3. 最后看 `examples.md`，确认接口和 CLI 命令。

## 6. 与接口案例的对照关系

这份文档负责解释“字段规则为什么这样设计”，不负责代替接口清单。

当 Skill 需要给出可执行结论时，建议固定这样引用：

1. 先用本文件解释 `usageType` 和 `systemView` 的区别。
2. 再到 `examples.md` 查看表、字段、行接口的真实方法、路径、入参、出参。
3. 如果要判断当前 Skill 链路里哪些能力已经可以直接执行，回看当前技能目录下的 `SKILL.md` 和 `build-flow.md`。

特别注意：

- 系统视图规则本身不是 CLI 命令。
- 相关事实通常体现在项目工作流绑定、表详情、字段配置或项目菜单配置中。
- 说明时要明确“规则来源”和“接口落点”是两件事。
