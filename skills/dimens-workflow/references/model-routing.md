# dimens-workflow 模型路由说明

## 1. 这份文档解决什么问题

这份文档用于提醒 AI：工作流模型配置不能只凭设计文档推断，必须区分“文档结论”和“当前实现”。

## 2. 文档结论

从产品设计口径看：

- 团队级有默认模型配置
- 节点未显式指定模型时，理想上可以回退到默认模型

## 3. 当前实现口径

基于当前 Skill 体系已确认的实现结论：

- 团队默认模型管理页面已存在
- `POST /app/flow/:teamId/v1/chat/completions` 已支持默认文本模型模式
- 但普通工作流内部 LLM 节点，不应直接假设一定会自动吃到 `default_models`

## 4. 提示语写法建议

写相关 Skill 时建议使用以下口径：

- “聊天兼容接口已支持团队默认文本模型模式”
- “普通工作流节点是否自动回退默认模型，需要按当前实现进一步确认，不能直接假设”

## 5. 什么时候必须提醒用户这个差异

- 用户说“不是已经有默认模型了吗，为什么节点还报错”
- 用户把 `chat/completions` 成功调用经验直接套到工作流节点上
- 用户准备做统一模型策略说明文档
