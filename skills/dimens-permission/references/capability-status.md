# dimens-permission 能力状态说明

## 1. 文档目标

这份文档专门回答：

`dimens-permission` 里哪些能力当前只是 server 真实接口，哪些并不是 CLI 已封装能力。

如果不把这层写清楚，Skill 很容易把“后端有接口”误说成“CLI 已可直接操作”。

---

## 2. 三种状态口径

| 状态 | 含义 | 输出要求 |
| --- | --- | --- |
| `已封装` | 当前已有 CLI 命令 | 可以直接给命令案例 |
| `server-only` | 当前只有后端接口或服务入口 | 只能给接口案例，不能伪装成 CLI 能力 |
| `部分对齐` | CLI 有相关上下游能力，但不能等同于完整权限管理能力 | 必须提示边界 |

---

## 3. 当前权限能力状态

### 3.1 server-only

当前 `dimens-permission` 主体能力基本都属于 `server-only`：

| 能力 | 当前状态 | 说明 |
| --- | --- | --- |
| 项目权限管理接口 | `server-only` | CLI 没有直接封装权限增删改查命令 |
| 行级策略管理接口 | `server-only` | CLI 没有直接封装策略管理命令 |
| 协同权限快照与广播过滤 | `server-only` | 主要是后端服务与 WebSocket 链路能力 |

### 3.2 部分对齐

下面这些能力容易被误判成“权限 CLI 已支持”：

| 现象 | 为什么只是部分对齐 |
| --- | --- |
| `project list/info`、`sheet info` 能查出结果 | 只能说明上下文和资源读取入口存在，不等于权限管理能力已 CLI 化 |
| `row page/info` 能读出数据 | 只能说明某次读取命中了当前权限，不等于已经有权限诊断命令 |
| `row set-cell`、`row update` 提交被拒绝 | 说明后端在做权限裁决，但 CLI 并没有权限管理接口本身 |

---

## 4. Skill 输出要求

当用户问“这个权限能力 CLI 有没有”时，建议固定按下面顺序回答：

1. 先说明当前属于 `已封装 / server-only / 部分对齐` 哪一类。
2. 如果没有 CLI，就给真实接口案例，不要伪装成命令能力。
3. 如果只是部分对齐，要明确这是“上下游能力存在”，不是“权限管理已命令化”。

---

## 5. 与其他 references 的关系

建议按下面顺序组合使用：

1. 先看 `examples.md`，确认真实接口入口。
2. 再看本文件，确认这项能力到底有没有 CLI。
3. 如果是具体业务场景，再看 `scenario-routing.md`。
